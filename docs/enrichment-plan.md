> 基准代码:`/Users/frankcj/workplace/new-typing-game/index.html`(单文件,1254 行,vanilla JS + Canvas,无构建步骤)。
> 本方案以「体验向」提案 B 为骨架(三位评审中两位推荐为基底、平衡审计全数复核通过),嫁接提案 A 的双流 per-level RNG、Boss 长词覆盖、精准打字奖励,以及提案 C 的 addGold()/spend() 记账纪律与「余金换分」收尾;并逐条修复了评审提出的全部红旗(30 WPM 上限口径、Volley 稀疏场衰减、V3 价格档口、T5→T6 / T7→T8 词长台阶过小、单流 RNG 无法复现重试、箭矢命中判定顺序等)。文中所有词长均值、词库计数均已对真实词表重新实测(词库实为 **3,682** 词,而非简介所说 ~2400;长度分布 3:280 / 4:808 / 5:925 / 6:598 / 7:505 / 8:358 / 9:166 / 10:38 / 11:3 / 12:1)。

## 1. 总览

Longbow 从「无尽刷分」改造为一场 **100 关战役**:每关是一次有限配额的法军突击(sortie),打完清场即过关;每 10 关为一档(共 10 档),档尾是一位有名有姓的法军勇士(Boss)攻城战。难度沿两条硬轴同时上升——**词长实测均值从 3.74 个字母升到 8.11**(满足需求 2),**每分钟来敌从 18 严格单调升到 40**(满足需求 3,每个档位边界至少抬高「出兵频率」或「单波人数」之一)。因为打长词必然降低射速(45 WPM 下射速从 T1 到 T10 天然下降约 47%),后期压力的数学出口是弓手升级:**Bodkin 伤害轨(1→4)+ Volley 齐射轨(每 N 箭附赠一箭)**,让「打出一个 9 字母长词 = 两名板甲骑士同时倒下」成为后期的核心爽点。塔升级(石工,100→420 HP)与可重复购买的修缮是慢速打字者的泄压阀。节奏是刻意设计的「锯齿」:每档开局略烫(负载比可短暂触及 1.05–1.15),Boss 赏金恰好资助下一件大件,买完立刻在下一章感到变强。存档按关卡设检查点,重试同关时敌人脚本与词序完全复现(同种子=可练习),死亡不再浪费时间。种子可复现性(`?seed=`)完整保留并强化为「同种子 = 同一场 100 关战役」。

## 2. 关卡结构

**「关」的定义。** 关卡号 `L ∈ 1..100`,档位 `T = ceil(L/10)`(1..10),档内序号 `ℓ = ((L-1)%10)+1`。一关是一段**有限、预生成、种子确定**的出兵脚本:总配额 `Q(T)` 名敌人(12→34,见总表),按档位间隔与波次分布出兵,右侧刷出、走向塔。删除现有的 `KILLS_PER_LEVEL` 涌现式升级(index.html 第 362 行常量与 `hitEnemy()` 内 551–558 行的升级块),`level` 变为显式战役计数器。

**关内节奏。** 出兵间隔 = `STAGES[T].interval × ramp(ℓ) × push × jitter`:档内爬坡 `ramp(ℓ) = 1.05 − 0.015×(ℓ−1)`(每档第 1 关 ×1.05 → 第 9 关 ×0.93,向 Boss 收紧);**冲锋段** `push = 0.85`(配额最后 25% 出兵提速,让每关都有一个浪头);抖动 `jitter = 0.85 + 0.3×rngSpawn()`(比现在的 0.75–1.25 收窄,波次更可读,仍走种子流)。普通关时长约 55–75 秒。

**过关条件:** `spawnedCount === Q && enemies.length === 0`(Boss 关另需 Boss 死亡)。无时间上限——时间压力就是行走速度本身(战场 ~810 px,Cotton 13 秒抵塔、Plate 22.5 秒,行走速度与撞击伤害全程**不**缩放,保持可读性)。

**关间流程(全键盘):** 最后一杀 → 0.6 秒停顿 → 号角 + 「Level 23 cleared · +45 gold」横幅 → 商店面板(打字购买,见 §6)→ 输入 `onward` + SPACE 进入下一关 → 关卡横幅 + 2 秒宽限后首波出兵。塔 HP **不**在普通关之间自动回满——它是贯穿全程的战略资源,修缮要花金币;仅 Boss 关开局免费修满(守军加固城墙的演出)。

**失败与重试:** 塔归零 → 失败画面 → ENTER 重试**同一关**:塔 HP 恢复为 `max(进关时HP, 40%×上限)`(防死亡螺旋),金币 = 进关快照 + 本次失败尝试击杀金的 **50%**(有缓冲但每分钟收益严格劣于通关,不可刷)。同关重试的出兵脚本与词序**完全相同**(见 §9)——死亡变成可学习的练习(「第 40 秒会来一队三板甲」)。输入 `abandon` + SPACE 放弃战役重新开始。

**检查点与存档:** 每次过关、每笔购买、每次 Boss 击破自动写入 localStorage(§9)。战役全程约 90–110 分钟,支持多次坐下续玩:启动页显示「ENTER — continue at Level 37 (412 gold)」。

**第 100 关之后:** 胜利画面「Agincourt Stands / England Stands」——战役统计(用时、准确率、箭数)+ **剩余金币按 1:1 换算入最终分数**(提案 C 的收尾闭环,囤金有意义)。随后解锁 **Endless March(无尽征途)**:第 101+ 关复用 T10 行参数,间隔每关 ×0.97(下限 1.5s)、配额每关 +2、每 10 关一个 Boss(HP ×1.15 复利);无新升级可买,是纯打字技巧的冲分模式,最佳关数存档,同样按种子可分享复现。

## 4. 公式

以下可直接落入代码(新表命名 `STAGES`,避免与现有装甲表 `TIERS` 冲突——提案 C 的命名裁定):

```js
const tierOf = L => Math.ceil(L / 10);            // 档 T
const lvlIn  = L => ((L - 1) % 10) + 1;           // 档内序号 ℓ (1..10; ℓ=10 为 Boss 关)

// 每关敌人配额
const quota = T => (T <= 9 ? 10 + 2 * T : 34);    // 12,14,...,28,34

// STAGES[T-1] = { wordMin, wordMax, heavyShare, heavyMin, heavyMax,
//                 interval, groupDist, mix }
// interval: 3.3,3.1,3.3,3.4,3.4,3.6,3.8,3.9,3.8,3.3  (秒)
// groupDist 示例 T7: [[1,.45],[2,.40],[3,.15]]
// heavyShare: 0,0,0,0,0,.10,0,.10,0,.50
// heavy 池:  T6=8..9, T8=9..10, T10=8..12
// mix: 见 §3 表

// 抽词(池内均匀;拒绝 C 的“先均匀抽长度”方案——11/12 字母仅 3+1 词,会导致
// 'nevertheless' 占 T10 两成抽取,评审判定不可实现):
function nextWord() {
  const s = STAGES[T - 1];
  let [mn, mx] = bossAlive
    ? [Math.min(9, s.wordMax), Math.min(12, s.wordMax + 3)]   // Boss 长词覆盖(§5)
    : (rngWords() < s.heavyShare ? [s.heavyMin, s.heavyMax]
                                 : [s.wordMin, s.wordMax]);
  // 池内均匀抽取,近 8 词环形去重(替换现有 lastWord 单词去重)
}

// 出兵间隔(startLevel 时用 rngSpawn 预生成整关脚本:[(t0,[tiers]),(t1,...)...])
gap = s.interval
    * (1.05 - 0.015 * (lvlIn(L) - 1))                 // 档内爬坡 ×1.05→×0.93
    * (spawned >= 0.75 * quota(T) ? 0.85 : 1.0)       // 冲锋段
    * (0.85 + 0.30 * rngSpawn());                     // 抖动
// 波内成员 x 坐标依次 +26px 错开;装甲按 mix 用 rngSpawn 抽取

// 敌人与战斗(现有 TIERS 的 hits/speed/dmg/score 全部不变)
e.maxHp = TIERS[e.tier].hits;                          // 1/2/3/4
const ARCHER_DMG  = [1, 2, 3, 4];                      // hitEnemy(): e.hp -= ARCHER_DMG[ranks.archer]
const VOLLEY_EVERY = [0, 4, 3, 2, 1];                  // 每第 N 箭附赠一箭 → 次近敌;
                                                       // 无次近敌时叠加射向最近敌(修复稀疏场衰减)
const towerMaxHP  = () => 100 + 40 * ranks.tower;      // 替换 TOWER_MAX_HP 常量(0..8 级)

// 经济
killGold   = [1, 2, 3, 4][e.tier];                     // 与 hits 同构
clearBonus = 10 + 5 * T;                               // 首次通关
bossBounty = 25 * T;
accuracyKicker = 5 * T;                                // 本关准确率 ≥97% 时(用现有 keystrokes/typos)
mendCost = 30; mendHeal = 30;                          // 可重复
```

## 5. Boss 关设计(每逢 ℓ=10)

每 10 关一场攻城战,是「布景 + 力量拐点」的双重节拍:

- **开场免费修满城墙**(「守军加固了城墙」横幅)——每场 Boss 战都从干净状态开打,结构性杜绝跛脚进档的死亡螺旋(评审最佳创意之一)。
- **Boss 本体:** 具名法军勇士,体型 ×1.7 绘制(`drawEnemy` 内 `ctx.scale`),独立顶部 HP 条 + 名字横幅;速度 30 px/s,撞塔 8 伤害/1.2s;HP 表:10/16/24/33/42/54/68/84/102/120(L10→L100)。赏金 `25×T`。
- **护卫涓流:** Boss 在场期间,以 2× 档位间隔刷单兵(档位装甲构成),最多 `Q/2` 名,Boss 死亡即停;保证混合压力而非纯打靶。
- **Boss 长词覆盖(嫁接自提案 A,评审一致推荐):** Boss 存活期间词库切换为 `[min(9, wordMax(T)), min(12, wordMax(T)+3)]` —— T1 为 4–7,逐档抬升,T6 起为 9–12(208 词,均值 9.23;评审裁定不用 A 原版 10–12 的 42 词小池)。11–12 字母词从此只在 Boss 战出现,Boss 成为「打字 Boss」而非血牛;档位仪式文案也不再虚假承诺常规关出现 11–12 字母词(修复评审红旗)。
- **战后档位仪式:** 「Tier 4 cleared — the Iron Vanguard is broken」+ 实测 WPM/准确率(现有计数器)+ 赏金结算 + 下一档预告(「法军将成对推进——单词增至 5–8 字母」)。**Boss 赏金刻意对准下一件大件**:L30→D3、L40→V1、L60→D4、L70→V2、L80→V3、L90→V4(见 §7)。
- **L100 终战核算(45 WPM,D4+V4):** Constable 120 HP = 30 箭 = 15 次双箭射击 ≈ 48s 纯打字(算上 ~35% 箭耗在护卫约 74s),对比预算 27s 行走 + 420/6.67 ≈ 63s 撞击 = 90s —— 紧、史诗、公平 ✓。

## 6. 升级系统

所有购买在关间商店内**打字完成**(复用完全匹配 + SPACE 的核心动词、绿字部分匹配渲染、`wordShake` + `sfx.error()` 的买不起反馈——零新输入概念,全程无鼠标)。商店显示金币、塔 HP/上限、各词条价格与已购等级圆点;买不起的条目灰显红价。无计时(喘息本身是节奏的一部分),输入 `onward` 出发。

**轨 1 — 塔:`masonry`(石工,8 级)** — 每级 **+40 上限并立即 +40 治疗**(购买当下就爽)。上限 100→140→180→220→260→300→340→380→**420**。价格 60/120/180/250/320/400/500/600(合计 **2,430g**),预期约每 10–12 关一级。代码:`towerMaxHP() = 100 + 40×ranks.tower` 替换第 360 行常量(`drawTower()` 第 813/819 行血条与 796/802 行裂纹阈值改读函数)。

**轨 2 — 弓手:`bodkin`(3 级)** — 箭伤 1→2→3→4(`hitEnemy()` 改 `e.hp -= ARCHER_DMG[ranks.archer]`)。价格 250/700/1,200(合计 **2,150g**),预期购于 ~L12/~L30/~L60。每级都对准装甲断点、肉眼可见地改变敌人死法(提案 A 的断点设计法):D2 皮甲 1 箭、D3 链甲 1 箭+板甲 2 箭、D4 板甲 1 箭。级名:Bodkin Points / Heavy Warbow / Master Fletcher。

**轨 3 — 齐射:`volley`(4 级,采纳的可选轨)** — 每第 4/3/2/1 箭同时向**次近敌**多射一箭(无次近敌时叠加最近敌——修复评审「稀疏场衰减」红旗);等效箭量 ×1.25/×1.33/×1.5/×2.0。价格 550/900/**1,150**/1,400(合计 **4,000g**;V3 由 1,250 降为 1,150,修复评审指出的仅 40g 档口余量)。**保留理由(非装饰):** D4 之后所有敌人都是 1 箭死,而 8+ 字母词把 45 WPM 的射速压到 ~21 发/分——Volley 是后期供给端唯一还能扩张的杠杆,同时是长弓「箭如飞蝗」的招牌幻想;没有它,T9–T10 在数学上不可解。终级名「Arrow Storm」。

**消耗品 — 修缮:`mend`** — 30g 换 +30 塔 HP,可重复,仅商店内。经济泄压阀:把打字失误变成金币税,让慢速打字者用盈余换生存,并在轨道封顶后维持金币意义。

**精准奖励(嫁接自提案 A):** 本关打字准确率 ≥97%(现有 `keystrokes/typos` 计数器,按关重置)加 `5×T` 金。**裁定:** 用它替换提案 B 的 flawless 奖励——两个并行的技巧奖励违反 YAGNI,评审两票偏好准确率版(~3 行代码,奖励干净输入而非退格连打)。不计入下表的可负担性预算。

## 8. 平衡性验算

供给 `shots/min = WPM×5 × 0.85 ÷ (L̄+1)`(L̄ 为实测池均值,+1 是发射的 SPACE,0.85 为反应/失误开销)× Volley 倍率;

## 9. 存档与随机种子

**localStorage 键 `longbow.save.v1`(带版本号供迁移):**

```js
{ v: 1,
  campaignSeed,                 // 战役创建时定死:Date.now() 或 ?seed= 覆盖(newSeed() 不变)
  nextLevel, gold,
  ranks: { tower, archer, volley },
  hpAtLevelStart,               // 进关快照 → 刷新页面 = 重试本关,无“刷新回血”漏洞
  bestLevel, victory, endlessBest,
  stats: { kills, arrows, keystrokes, typos, playtimeS } }
```

写入时机:过关、每笔购买、Boss 击破、失败(仅 bestLevel/stats)、胜利。localStorage 不可用(file:// 隐私模式)或损坏 → 内存兜底 + 一次性提示,绝不阻断游戏。

**种子模型(修复评审判定的 B 案单流缺陷):** `campaignSeed` 仅在新战役创建时掷定并存档,续玩绝不重掷;每关派生 `levelSeed = (campaignSeed ^ Math.imul(L, 0x9E3779B9)) | 0`,并生成**两条独立** mulberry32 流:`rngWords = mulberry32(levelSeed)`(抽词)、`rngSpawn = mulberry32(levelSeed ^ 0x85EBCA6B)`(出兵脚本在 `startLevel()` 一次性预生成)。两条流是必须的:抽词次数取决于玩家手速,单流会让出兵时序随打字快慢漂移——分流后,敌人脚本与词序各自只依赖 `(campaignSeed, L)`,**同关重试 = 真正的练习局**,`?seed=X` 可跨会话复现整场战役与任意报告。**裁定:** 弃用提案 C 的 attempt 计数盐(评审指出它同时扰动出兵脚本,自相矛盾;可练习性优先)。装饰性特效继续用 `Math.random()`。URL 带 `?seed=` 时开启**新**战役,但启动页保留 CONTINUE 选项直到玩家输入 `muster` 确认(不静默覆盖存档)。Endless 关沿用同一派生(L 继续递增),无尽冲分同样可按种子分享。

## 12. 明确不做的事(YAGNI)

- **敌人 HP 随档倍增(提案 A 的 ×act)** — 破坏「四色装甲 = 1/2/3/4 格血」的全程可读语言;压力已由数量、编队、词长承担。
- **暴击/随机伤害轨** — RNG 伤害毁掉整套 ceil(hp/D) 可读性与平衡账本。
- **Plunder 收入轨(提案 C)** — 平衡评审证明 250g 回报 ~1,500g,是支配性策略而非抉择。
- **每敌一词/多词道** — 那是另一款游戏;单词面板 + 自动瞄准就是 Longbow 的身份。
- **特效箭(火/冰)、连击/连胜计分表、成就、NG+/声望、元货币** — 与两条主轨重复或稀释账本。
- **难度模式** — 检查点 + 重试 + 修缮已是事实上的辅助机制;30 WPM 辅助模式(×1.2 间隔)记录在案但不做。
- **关卡选择与旧关重刷(提案 A)** — M4 之后的装饰,砍;重试规则已覆盖练习需求。
- **智能目标优先级** — 最近优先保留;漏兵分诊本身是技巧(playtest 尖叫再议)。
- **关内商店、塔自动回血、鼠标 UI、行走速度/撞击伤害缩放** — 分别破坏关内张力、修缮经济、键盘纯度、弹道解算手感与「几秒抵塔」的可读性。
- **C 的“先均匀抽长度”采样** — 词表在 11/12 字母仅 3+1 词,数学上不可行。
- **B 的 flawless 奖励** — 被精准奖励替代,不留两个并行技巧加成。

## 13. 风险与开放问题

1. **刀锋锚点(T6/T8–T10 负载比 0.93–0.96)** 依赖 0.85 开销常数,而失误代价随词长非线性增长。缓解:M5 机器人用词长相关开销(0.90→0.78)重验;三重独立泄压阀(塔级、修缮、重试地板)已内建;超标则单元格调 `STAGES`(±10%)或负载目标降至 0.88。
2. **Volley ×2 依赖次近敌存在**——孤敌叠加规则已兜底,但 T8–T9 关尾稀疏段仍需机器人专项验证;若仍塌,备选是让第二箭对同一目标 100% 生效(等效纯 ×2)。
3. **30 WPM 玩家 L21–50 撞墙是定位选择而非 bug**——必须在启动页与商店提示中说清(「军需官力劝换重箭头」),否则读作劣质平衡。
4. **9–12 字母 Boss 词池 208 词**,十场 Boss 战会有重复;可接受,若刺眼则为词表补 ~100 个 10–12 字母词(纯数据改动,顺带缓解 T10 重词池)。
5. **真人方差 vs 模型**:爆发/卡壳可能让 T8–T10 实际修缮开销高于 1,609g 盈余;缓解:盈余本身、420 塔上限、重试 50% 金;M5 数字锁定前需 2–3 次真人试玩对照机器人轨迹。
6. **确定性回归风险**:今后任何新随机消费必须声明走 `rngWords` 还是 `rngSpawn`,否则复现性静默破坏——在两条流声明处留强制注释,`__game` 提供首 8 个出兵事件的校验和。
7. **战役时长 ~100 分钟**:弃档率取决于关口存档手感,M4 不可延期;开放问题:是否在仪式屏加「本次坐下用时」提示鼓励分段游玩。
8. **存档作弊/刷金**:50% 重试金与一次性通关奖已封顶,最坏情形是缓慢无聊的金币——接受。
9. **开放问题**:Endless March 的 ×0.97 递减与 +2 配额未经机器人验证(胜利后内容,非发售阻塞);精准奖励 97% 阈值对长词档是否过苛,交由 M5 数据定夺。
